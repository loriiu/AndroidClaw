## 1. 整体架构设计

### 1.1 架构原则

AndroidClaw 采用 **Clean Architecture + MVVM** 架构，遵循以下原则：

| 原则 | 说明 |
|------|------|
| **关注点分离** | UI / 业务逻辑 / 数据存储分层 |
| **依赖倒置** | 外层依赖内层，内层不依赖外层 |
| **模块化** | 核心模块独立，便于测试和复用 |
| **响应式** | 使用 Flow 进行数据流转 |

## 3. 技术选型对比与决策

### 3.1 框架选型

| 技术 | 选型 | 备选 | 决策理由 |
|------|------|------|---------|
| **语言** | Kotlin 1.9.22 | Java | 现代语法、空安全、协程支持 |
| **UI** | Jetpack Compose | XML | 声明式、预览友好、性能好 |
| **DI** | Hilt | Koin/Dagger | 官方推荐、编译时校验 |
| **架构** | MVVM + Clean | MVP/MVI | 分层清晰、社区成熟 |
| **异步** | Coroutines + Flow | RxJava | Kotlin 原生、简洁 |
| **存储** | Room + DataStore | SQLite/SharedPreferences | 类型安全、Flow 支持 |
| **调度** | WorkManager | AlarmManager/Service | 可靠性、节电 |

### 3.2 LLM 集成方案对比

| 方案 | 优点 | 缺点 | 适用场景 |
|------|------|------|---------|
| **直接 HTTP** | 简单、轻量 | 需要处理流式、重试 | MVP 阶段 |
| **Spring AI Client** | 功能完整 | 依赖重、不适合 Android | 参考实现 |
| **Android AI SDK** | 官方支持 | 覆盖有限 | Google AI |

**决策**：采用直接 HTTP 调用 + Retrofit，参考 JavaClaw 的 Spring AI 实现逻辑自行适配。

### 3.3 任务调度方案对比

| 方案 | 优点 | 缺点 | 适用场景 |
|------|------|------|---------|
| **JobRunr** (JavaClaw) | 功能强大、Dashboard | Java 生态、不支持 Android | 参考 |
| **WorkManager** | 官方推荐、可靠性高 | 功能相对简单 | Android 首选 |
| **AlarmManager** | 精确计时 | 设备休眠问题 | 仅精确闹钟 |
| **Service + Handler** | 灵活 | 后台限制多 | 不推荐 |

**决策**：采用 WorkManager，利用 `PeriodicWorkRequest` 实现重复任务，`OneTimeWorkRequest` 实现一次性任务。

### 3.4 数据存储方案对比

| 方案 | 优点 | 缺点 | 适用场景 |
|------|------|------|---------|
| **H2** (JavaClaw) | 嵌入式、SQL 完整 | Java 生态 | 参考 |
| **Room** | 官方推荐、类型安全 | 功能相对 H2 少 | Android 首选 |
| **DataStore** | 异步、Proto 支持 | KV 存储 | 配置存储 |
| **EncryptedSharedPreferences** | 加密、安全 | API 级别 | 敏感数据 |

**决策**：Room 作为主数据库，DataStore 管理非敏感配置，EncryptedSharedPreferences 存储 API Key。

## 4. 数据流设计

### 4.1 对话流程

```
┌──────────┐     ┌───────────┐     ┌────────────┐     ┌─────────┐
│   User   │────▶│   ViewModel │────▶│    Agent    │────▶│   LLM   │
│  Input   │     │            │     │   Service   │     │   API   │
└──────────┘     └─────┬─────┘     └────────────┘     └─────────┘
                      │                    ▲
                      │                    │
                      ▼                    │
                ┌───────────┐              │
                │ Repository│◀─────────────┘
                └─────┬─────┘
                      │
                      ▼
                ┌───────────┐
                │   Room    │
                │  Database │
                └───────────┘
```

**详细流程**：
1. 用户在 ChatScreen 输入消息
2. ChatViewModel 调用 `AgentRepository.sendMessage()`
3. Repository 将消息保存到 Room 数据库
4. AgentService 构建 Prompt（包含系统提示、上下文、工具定义）
5. 调用 LLM API 获取响应
6. 解析响应，判断是否有工具调用
7. 如有工具调用，执行工具并继续
8. 最终回复保存到数据库
9. ViewModel 收集 Flow 更新 UI

## 7. 安全设计

### 7.1 安全层级

| 层级 | 措施 | 说明 |
|------|------|------|
| **传输层** | HTTPS/TLS 1.2+ | 网络通信加密 |
| **存储层** | AES-256 加密 | 敏感数据保护 |
| **权限层** | 最小权限原则 | 防止滥用 |
| **审计层** | 日志脱敏 | 不记录敏感信息 |

### 7.2 API Key 安全

1. **加密存储**：使用 Android Keystore 生成密钥，EncryptedSharedPreferences 存储
2. **内存安全**：不在日志中打印，不在全局变量中长期保存
3. **传输安全**：每次请求动态添加 Authorization 头
4. **过期机制**：支持 Key 轮换

### 7.3 隐私保护

| 场景 | 保护措施 |
|------|---------|
| **本地存储** | 敏感数据加密，数据库不存储明文 |
| **网络传输** | HTTPS，证书校验 |
| **日志记录** | 脱敏处理，不记录对话内容 |
| **权限申请** | 运行时申请，说明用途 |
| **数据删除** | 支持清除所有本地数据 |

### 8.2 状态管理

```kotlin
// UI State
data class ChatUiState(
    val messages: List<Message> = emptyList(),
    val isLoading: Boolean = false,
    val error: String? = null
)

// ViewModel
class ChatViewModel @Inject constructor(
    private val agentRepository: AgentRepository
) : ViewModel() {
    
    private val _uiState = MutableStateFlow(ChatUiState())
    val uiState: StateFlow<ChatUiState> = _uiState.asStateFlow()
    
    fun sendMessage(text: String) {
        viewModelScope.launch {
            _uiState.update { it.copy(isLoading = true) }
            try {
                val response = agentRepository.sendMessage(text)
                _uiState.update { 
                    it.copy(
                        messages = it.messages + response,
                        isLoading = false
                    )
                }
            } catch (e: Exception) {
                _uiState.update { 
                    it.copy(error = e.message, isLoading = false)
                }
            }
        }
    }
}
```

### 9.1 工具调用机制

```kotlin
// 工具定义
interface Tool {
    val name: String
    val description: String
    suspend fun invoke(params: Map<String, Any>): ToolResult
}

// 任务工具
class TaskTool @Inject constructor(
    private val taskManager: TaskManager
) : Tool {
    override val name = "create_task"
    override val description = "创建一个新任务"
    
    override suspend fun invoke(params: Map<String, Any>): ToolResult {
        val name = params["name"] as String
        val description = params["description"] as String
        taskManager.createTask(name, description)
        return ToolResult("Task created: $name")
    }
}
```

    suspend fun listTools(): List<McpTool>
    suspend fun callTool(toolName: String, params: Map<String, Any>): JsonRpcResponse
}

// MCP 工具适配器
class McpToolAdapter(
    private val mcpClient: McpClient,
    private val tool: McpTool
) : Tool {
    override val name = tool.name
    override val description = tool.description
    
    override suspend fun invoke(params: Map<String, Any>): ToolResult {
        val response = mcpClient.callTool(name, params)
        return ToolResult(response.result)
    }
}
